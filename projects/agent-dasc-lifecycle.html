1. Concept & Google DS-STAR Foundations

Agent-DASC (Data Analytics Software Consultant) is an autonomous data science agent modeled on the core architecture of the **Google DS-STAR** research paper (*"DS-STAR: Data Science Agent via Iterative Planning and Verification"*).

Rather than relying on single-shot LLM queries to build scripts, DS-STAR establishes a structured multi-agent loop that mimics human data scientists. It automates analysis over heterogeneous data formats, designs step-by-step query programs, executes them in sandboxes, and validates outputs using rigorous verification checks.

2. Multi-Agent Pipeline Mechanics

Following Google's DS-STAR framework, Agent-DASC divides processing duties across five specialized agent components:

  1. Analyzer: Inspects heterogeneous datasets (CSV, JSON, Excel, markdown) to extract structure and statistics.
  2. Planner: Formulates a structured, multi-step logical execution plan.
  3. Coder: Translates plan items into executable Python, generating SQL operations and analysis code.
  4. Debugger: Captures execution errors in sandbox loops and performs automatic refactoring.
  5. Verifier: Act as an LLM-based judge to evaluate output validity against the original requirements, refining the plan if the score falls below threshold parameters.

3. Implementation Details

The implementation of the DS-STAR-based planning and verification agent uses structured schemas to validate code executions. Below is a code block showing the execution wrapper: 

class DSStarAgent:
    def __init__(self, database, model):
        self.planner = PlannerAgent(model)
        self.coder = CoderAgent(model)
        self.verifier = VerifierAgent(model)

    def solve_query(self, user_query):
        plan = self.planner.generate_plan(user_query)
        result = self.coder.execute_steps(plan)
        
        # Iterative verification check
        verdict = self.verifier.judge(result, user_query)
        if verdict["status"] == "FAILED":
            plan = self.planner.refine_plan(plan, verdict["reason"])
            result = self.coder.execute_steps(plan)
        return result

4. Documenting Product Lifecycle (SOPs)

The repository's docs/product-lifecycle branch maps the Standard Operating Procedures (SOPs) required to maintain and evaluate agent workflows in production:

  • Prompt Governance: Tracking system prompts and few-shot stores through pipeline versioning.
  • Relational DB Mapping: Schema embedding generation to locate correct columns across databases containing hundreds of tables.
  • Drift Monitoring: Auto-evaluation of verifier accuracy against expert-approved baseline datasets.
